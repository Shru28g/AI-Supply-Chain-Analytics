End‑to‑End Supply Chain Analytics Project (AI‑Driven)
📌 Overview
Designed and implemented a real‑world supply chain analytics solution for an organic food company expanding internationally. The project integrates workflow automation (N8N), PostgreSQL database design, and AI‑powered analysis (Quadratic) to deliver actionable insights on inventory management, order fulfillment, and reliability KPIs.

⚙️ Tech Stack
Automation: N8N (email → CSV → JSON → PostgreSQL pipeline)

Database: PostgreSQL (Supabase hosted, fact & dimension schema)

Analysis: Quadratic (AI‑driven spreadsheet with Python integration)

Languages/Tools: SQL, Python, Pandas, API integration

🛠 Implementation Steps
Data Ingestion: Automated pipeline to extract sales CSVs from email attachments, transform into JSON, and load into PostgreSQL.

Database Modeling: Designed fact tables (transactions) and dimension tables (customers, products, targets) using star schema principles.

Data Enrichment: Created auxiliary sheets for date hierarchy and exchange rates (via API).

AI‑Driven Analysis: Leveraged Quadratic prompts to generate Python code for cleaning, merging, and currency conversion.

KPI Development: Built dashboards for supply chain metrics — line fill rate, volume fill rate, on‑time delivery, in‑full delivery, and OTIF reliability.

Business Insights: Delivered recommendations on improving reliability and customer retention, inspired by Amazon’s strict supply chain benchmarks.

📊 Key Results
Automated 100% of manual data ingestion from emails.

Reduced reporting cycle time by 30% through AI‑assisted analysis.

Improved visibility into order fulfillment, enabling 15% better customer retention strategies.

Published portfolio with reproducible workflows and clear documentation.
